26 ## BSP-Module
27 Modul | Zweck
28 --------------- | ---------------------------------------------------------------------------------------------------------------------
29 ADC | Funktionen zur Verwendung des PIC-internen Analog-Digital-Konverters.
30 LCD | Ansteuerung des DOGM162-LCDs. Mit kleinen Anpassungen auch für andere Controller und Displays verwendbar.
31 UART | Funktionen fürs Lesen und Schreiben über die UART-Schnittstelle,
32 I2C | Lese- und Schreibfunktionen für die interne I2C-Schnittstelle.
33 LM75 | Ansteuerung des Temperatursensors TCN75, welcher protokollkompatibel zum Modell LM75 ist.
34 EEPROM | Verwendung des PIC-internen EEPROMS.
35 HELPERS | Hilfs-Makros und Funktionen wie Bit setzen / löschen, Bit toggle.
36 INTERRUPT | Globale Interruptroutine. Möglichkeit zum Hinzufügen von Callbackfunktionen für Timer-, UART- und externe Interrupts.
37 LOOPDELAY | Timer-basiertes Delay. Verwendung in Zustandsmaschinen.
38 EVENTS | Funktionen zur Verwendung von zeitlichen- und Tasterevents. Anwendung in Zustandsmaschinen.
39 MENU | Funktionen zur Darstellung eines Menüs auf zeichenbasierten LCDs. Verwendbar in Zustandsmaschinen.

42 ## Einbinden des BSP in ein MPLABX Projekt
43 1. Neues MPLABX Projekt erstellen
44 2. Die beiden Ordner "src" und "inc" in den Projektordner kopieren
45 3. Im Projekt Rechtsklick auf Source Files, danach "Add existing Item"
46 4. Im Browse-Dialog in den Source Ordner navigieren, alle C-Files anwählen und diese dem Projekt hinzuzufügen
47  **WICHTIG!! Die Auswahl "Store path as:" muss auf "Relative" eingestellt sein**
48 5. Schritte 3&4 wiederholen für die Include Files
49 6. Neues C-File erstellen und im Ordner "src" abspeichern
50 7. Zur Verwendung der Library muss folgende Code-Zeile im neu erstellten C-File hinzugefügt werden:

53  #include "../inc/PBA_config.h"
54  ```

57 ## BSP-Konfiguration
58 Zur Konfiguration muss die Header-Datei ```PBA_config.h ``` editiert werden.
59 Durch ein- oder auskommentieren der defines im Abschnitt "BSP-Konfiguration" lässt sich
60 das BSP auf die Bedürfnisse anpassen.
61  ```C
62 /******************** BSP-Konfiguration ***********************************************/
63 
64 #define PBA6/**<LCD wird für 3.3V initialisiert. Auskommentieren, falls nicht PBA6 verwendet wird*/
65 
66 #define USE_ADC_LIBRARY /**< ADC-Support Funktionen*/
67 
68 #define USE_LCD_LIBRARY /**< LCD DOGM162 Funktionen*/
69 
70 #define USE_UART_LIBRARY /**< UART Funktionen*/
71 
72 #define USE_I2C_LIBRARY /**< I2C Funktionen*/
73 
74 #define USE_LM75_LIBRARY /**< Ansteuerung I2C-Temperatursensor*/
75 
76 #define USE_EEPROM_LIBRARY /**< Verwendung internes EEPROM*/
77 
78 #define USE_HELPERS_LIBRARY /**< Nützliche Makros und Funktionen*/
79 
80 #define USE_INTERRUPT_LIBRARY /**< Globaler Interrupt-Handler*/
81 
82 #define USE_LOOPDELAY_LIBRARY /**< Timerbasiertes Delay*/
83 
84 #define USE_EVENTS_LIBRARY /**< Zeit- und Taster-Events*/
85 
86 #define USE_MENU_LIBRARY /**< Funktionen Erstellung und Darstellung eines Menus auf LCD*/
87  ```

101 ### STDOUT
102 Printf-Ausgaben können entweder ans LCD oder über UART gesendet werden. Mittels Compiler-Schalter
103 kann zwischen den beiden Varianten umgestellt werden. Zu beachten ist, dass das jeweilige BSP-Modul
104 aktiviert sein muss. Sind weder UART- noch LCD-Modul aktiviert, müssen beide Compiler-Schalter deaktiviert werden.
105  ```C
106 /********** Auswahl STDOUT ****************************************************/
107 #define STDOUT_LCD true /**< Ausgabe printf auf LCD*/
108 //#define STDOUT_UART true /**< Ausgabe printf auf UART-Schnittstelle*/
109  ```

112 ## Interrupts
113 Der Mikrocontroller verfügt nur über einen Interrupt-Vektor. Die globale Interrupt-Routine
114 ist in der Datei ```PBA_Interrupts.c ``` umgesetzt. Für die wichtigsten Peripherie-
115 Interrupts können Callback-Funktionen hinzugefügt werden. Folgende Funktionen stehen dazu zur Verfügung:
116  ```C
117 uint8_t INT_AddTmr0CallbackFnc (callbackFunction_t p_callBackFunction);
118 uint8_t INT_AddTmr1CallbackFnc (callbackFunction_t p_callBackFunction);
119 uint8_t INT_AddTmr2CallbackFnc (callbackFunction_t p_callBackFunction);
120 uint8_t INT_AddUartCallbackFnc (callbackFunction_t p_callBackFunction);
121 uint8_t INT_AddExtIntCallbackFnc (callbackFunction_t p_callBackFunction);
122 ```
123 Pro Timer können jeweils drei Funktionen hinterlegt werden. Die restlichen Interrupts akzeptieren nur eine
124 Callbackfunktion. Über den Rückgabeparameter kann überprüft werden, ob die Funktion erfolgreich hinterlegt wurde.
125 
126 Rückgabewert | Bedeutung
127 --------------- | -----------------------------------------------------------
128 0 | Erfolg
129 1 | Fehler, die maximale Anzahl Funktionen ist bereits erreicht

148 ## Verwendung der Loopdelay-Library
149 Nach der Initialisierung der Hardware muss das Loopdelay initialisiert werden.
150 Dafür wird die Funktion ```void LOOPDELAY_Init(uint16_t loopDelayTimeMS);``` aufgerufen,
151 wobei der Übergabeparameter dem gewünschten Loopdelay in Millisekunden (ms) entspricht.
152 Ausgeführt wird das Delay durch die Funktion ```void LOOPDELAY_Wait(void);```.
153 Der Benutzer ist selber verantwortlich, dass der gesamte Programmdurchlauf nicht länger als
154 die angegebene Zykluszeit dauert.
155 
156 ```C
157 #include "../inc/PBA_config.h"
158 void main(void)
159 {
160  PBA_Init(); /*Initialisieren der Hardware*/
161  LOOPDELAY_Init(10); /*Loopdelay Init, loop-time=10ms*/
162  while(1)
163  {
164  LOOPDELAY_Wait(); /*Zykluszeit 10ms*/
165  LED0^=1; /*LED0 blinkt mit einer Frequenz von 50Hz*/
166  }
167 }
168  ```

171 ## Verwendung der Event-Library
172 Die Event-Library bietet die Möglichkeit mit verschiedenen Events zu arbeiten.
173 Folgende Events sind definiert:
174 
175 Eventname | Funktion
176 ---------------------- | -------------------------------------------------------------
177 posEdge | Abfragen einer positiven Tasterflanke
178 negEdge | Abfragen einer negativen Tasterflanke
179 ActiveUntilMS(time) | Nach dem ersten Aufruf nur während der angegebenen Zeit aktiv
180 TimeoutMS(time) | Nach dem ersten Aufruf immer nach der angegebenen Zeit aktiv
181 
182 Zur Initialisierung der Bibliothek wird die Funktion ```void EVENTS_Init(void *p_state,events_t *p_events);``` aufgerufen. Der Übergabeparameter ```void *p_state``` ist der Pointer auf die in einer Aufzählung (enum) definierten states.
183 Der zweite Parameter ist der Pointer auf eine Struktur vom Type events_t. Über diese Struktur können Events erstellt und abgefragt werden.
184 Das Loopdelay wird über die bekannte Funktion ```void LOOPDELAY_Wait(void);```aufgerufen.
185 Zum Detektieren der Tasterflanken muss die Funktion ```EVENTS_Update(void);``` zyklisch ausgeführt werden. <br>
186 **Für zeitliche Events dürfen nur Werte die das Ein- oder Mehrfache der Zykluszeit betragen verwendet werden!**
187 
188 ```C
189 #include "../inc/PBA_config.h"
190 
191 #define AUS 0
192 #define EIN 1
193 
194 #define LEDANZEIGE LATD
195 
196 enum {ST_TIMEOUT,ACTIVE_UNTIL, ST_POSEDGE_NEGEDGE}state=ST_TIMEOUT; /*Aufzählung aller Zustände*/
197 
198 events_t events; /*Event-Struktur*/
199 
200 void main(void)
201 {
202 
203  PBA_Init(); /*Initialisierung Hardware*/
204  LOOPDELAY_Init(10); /*Initialisierung Loopdelay*/
205  EVENTS_Init(&state,&events); /*Initialisierung Events*/
206 
207  LED0=EIN;
208  while(1)
209  {
210  LOOPDELAY_Wait(); /*Zykluszeit 10ms*/
211  EVENTS_Update(); /*Events aktualisieren*/
212  switch(state)
213  {
214  case ST_TIMEOUT:
215  if(events.TimeoutMS(3000)) /*LED0 leuchtet während 3 Sekunden*/
216  {
217  LED0=0;
218  state=ST_ACTIVE_UNTIL;
219  }
220  break;
221  case ST_ACTIVE_UNTIL:
222  if(events.ActiveUntilMS(2000))
223  {
224  if(events.TimeoutMS(100))
225  {
226  LED0^=1; /*LED0 blinkt mit 5Hz für 2s*/
227  }
228  }
229  else
230  {
231  LED0=1;
232  state=ST_POSEDGE_NEGEDGE;
233  }
234  break;
235  case ST_POSEDGE_NEGEDGE:
236  if(events.posEdge.switch0) /*Bei jeder Tasterbetätigung wird die */
237  { /*Ledanzeige um 1 inkrementiert*/
238  LEDANZEIGE++;
239  }
240 
241  if(events.negEdge.switch7) /*Beim loslassen des Tasters */
242  {
243  LEDANZEIGE=1; /*zurücksetzen der Anzeige*/
244  state=ST_TIMEOUT; /*Rückkehr zum 1. Zustand*/
245  }
246  break;
247  }
248  }
249 }
250 ```
251 ## COM-Port Treiber
252 Je nach Betriebssystem und Konfiguration wird der Treiber für den USB-Seriell-Wandler FT232R nicht automatisch installiert.
253 Die Virtual Com Port (VCP) Treiber können im <a href="http://www.ftdichip.com/Drivers/VCP.htm" target="_blank">Downloadbereich</a> von FTDI Chip heruntergeladen werden. <br>
254 Für ungeübte Benutzer stellt der Hersteller <a href="http://www.ftdichip.com/Support/Documents/InstallGuides.htm" target="_blank">Installationsanleitungen</a> für alle gängigen Betriebssysteme zur Verfügung.
